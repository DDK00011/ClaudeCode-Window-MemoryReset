# 작업 일지 — 2026-05-23 (v1.3.0: 선별 종료 + 좀비 분석 + Round 2 보안 hardening)

> v1.3.0 — 활성 세션 보존 + 좀비 식별 + UAC injection / whitelist bypass 보안 패치.
> v1.2.x 까지의 "전부 종료" 전략을 사용자 요구로 "선별 종료" 로 전환.

## 동기

### 사용자 환경에서 발견된 실제 패턴

DDR4 64 GB / Ryzen 5 5600G / Windows 10 LTSC 환경에서 측정:

| 항목 | VS Code (활성 9) | Antigravity (활성 7) |
|---|---|---|
| 총 프로세스 | 23 | 21 |
| 총 메모리 | 4,815 MB | 4,587 MB |
| **claude.exe 개수** | **13** | **10** |
| **추정 잉여(좀비)** | **4** | **3** |
| GPU 프로세스 | 178 MB | **447 MB (+151%)** |
| 핸들/proc | 467 | **758 (+62%)** |

핵심 발견:
1. **claude.exe 본체가 진짜 메모리 범인** — 평균 350 MB × 23개 = **8 GB**. 손자 node.exe (MCP server, 60 MB × 22) 는 부차적.
2. **활성 IDE 세션 수와 claude.exe 개수가 불일치** — IDE 가 채팅 탭을 닫아도 자식 claude.exe 가 종료 안 됨. Windows `TerminateProcess` 의 non-cascade 한계 + IDE 확장의 짧은 cleanup timeout.
3. **60 GB 누수 시나리오 검증** — 350 MB × 150 + node 60 MB × 150 + IDE = ~66 GB → 사용자 보고와 일치.

### 기존 v1.2.x 의 한계

v1.2.x 는 **모든 매칭 프로세스 일괄 종료** — 사용자가 작업 중인 세션도 함께 사라짐.
사용자 요구: **"활성 세션은 살리고 불필요한 것만 지운다"**.

### P0: 선별 종료 3종 옵션

| 옵션 | 동작 | 사용 케이스 |
|---|---|---|
| `-KeepPids "1234,5678"` | 명시 PID 보존, 나머지 종료 | 활성 PID 를 이미 알고 있음 |
| `-Interactive` | 종료 전 PID 별 보존 입력 프롬프트 | PID 모름, 한 번에 결정하고 싶음 |
| `-OrphansOnly` | 부모 IDE 죽은 claude.exe/node.exe 만 | 안전 모드, 가장 보수적 |

내부 구조:
- `ConvertFrom-KeepPidsString` (line 343): CSV 문자열 → `[uint32[]]` 안전 파서.
- `Test-IsClaudeOrphan` (line 332): 부모 PID 죽음 + PID 재사용 두 케이스 모두 판별.
- `Get-TargetProcesses` 시그니처 확장 (line 359): `-ExcludePids` + `-OnlyOrphans` 파라미터.
- UAC 라운드트립 인자 전달 갱신 (line 114-125).

### P2: 좀비 분석 (`-Diagnose` 강화)

`Show-ZombieAnalysis` (line 612) — `claude.exe` 를 `ParentProcessId` 별로 그룹화:
- **[OK]**: 부모가 IDE/legitimate spawner (`Code|Antigravity|claude|cursor|windsurf|cmd|pwsh|bash|...`) → 활성
- **[DEAD]**: 부모 PID 가 enumerate 안 됨 → 확실한 좀비
- **[REUSED]**: 부모 PID 존재하지만 IDE 가 아님 (PID 재사용) → 좀비

요약 끝에 "안전 종료 명령" 복사용 출력:
```
[!] 확실한 좀비: N개 / M MB
    → 안전 종료:  .\MemoryReset.ps1 -OrphansOnly
```

## 검토 라운드 (Phase B — 검증)

### Round 1: 통합/엣지 케이스 검토 (general-purpose agent)
- **P0 없음** — 데이터 손실/wrong-kill 시나리오 발견 안 됨.
- P1 5건: null array unwrap, `-Interactive`+`-DryRun` silent skip, race window, 등.
- 트레이 데몬 (`MemoryReset-Tray.ps1:140-148`) 회귀 없음 확인.
- `.bat` 런처 4종 (`Run.bat`, `Run-DryRun.bat`, `Run-Deep.bat`, `Run-Diagnose.bat`) 기존 동작 보존 확인.

### Round 2: 보안 검토 (general-purpose agent, adversarial)
**P0 보안 결함 3건 발견** — 커밋 보류 결정:

1. **`-KeepPids` UAC injection** — 입력 검증 없이 elevated PowerShell 명령행에 전달.
   PoC: `-KeepPids '1234"; Start-Process calc.exe; #'` → 임의 코드 실행 가능.
2. **화이트리스트 경로 unanchored** — `\\\.vscode\\extensions\\...` 가 사용자 쓰기 가능한
   `C:\evil\.vscode\extensions\anthropic.claude-code-evil\claude.exe` 도 매칭 → admin 으로 임의 프로세스 종료 트리거.
3. **CLI 시그니처 단독 신뢰** — `--output-format stream-json` 만으로 매칭 → 위장 가능.

P1 4건: `ConvertFrom-KeepPidsString` overflow / unwrap, `Test-IsClaudeOrphan` 의 spawner 화이트리스트 부족, 등.
